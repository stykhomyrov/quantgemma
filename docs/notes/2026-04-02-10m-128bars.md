# 10m bars, 128-bar sequences from scratch

**Date:** 2026-04-02
**Status:** Completed — first positive result

## Setup

- Trained from base Gemma 3 270M
- 10m bars, SIGMA_WIN=144, all 748 symbols
- SEQ_BARS=128 (vs 64 previously), ~21 hours of context per sequence
- Data: v4, 918K train / 114K val / 114K test sequences
- LR=3e-5, BATCH_SIZE=8
- 1 hour training, 8500 steps
- seq_len=388 tokens (vs 194 with 64 bars)

## Results

- val_loss: 1.9849
- train_loss: 1.9910
- **Directional accuracy: 50.94% (+0.94% edge)**
- Clear context length effect

## Context length analysis

| Bars    | Accuracy | Edge   |
|---------|----------|--------|
| 0-9     | 50.18%   | +0.18% |
| 10-19   | 50.81%   | +0.81% |
| 20-29   | 50.82%   | +0.82% |
| 30-39   | 50.95%   | +0.95% |
| 40-49   | 50.99%   | +0.99% |
| 50-63   | 51.00%   | +1.00% |

## Takeaway

- Longer sequences are key — 128 bars shows real edge while 64 bars showed nothing
- Context length clearly helps: later bars in the sequence are more accurate
- The model needs ~10+ bars of context before it starts outperforming random
- val_loss is nearly identical to v2 (1.985) but directional accuracy is much better
- This confirms that loss alone doesn't predict directional performance
- Next: try even longer sequences (256 bars) or more training time
