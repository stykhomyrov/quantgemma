# QuantGemma

The purpose of this project is to develop a profitable trading strategy based on signalges from finetuned [Gemma 3 270M](https://huggingface.co/google/gemma-3-270m) compact LLM using tokenized market data. Tokenization approach is described in [docs/tokens.md](docs/tokens.md). The idea is simple: turn market data into a kind of "language" and use LLM to predict market moves. Experiments showed that it makes sense and finetuned model has an edge and is able to generate profits. Our goal is to increase these profits.

## Data

Raw data is stored in the directory `data/binance_futures` in Parquet files (e.g. `data/binance_futures/BTCUSDT/BTCUSDT-1m-2020-01.parquet`). Each file contains data for a single product for a single month and has following columns:

| Column name               | Column type              |
|---------------------------|--------------------------|
| open_time                 | timestamp with time zone |
| open                      | double                   |
| high                      | double                   |
| low                       | double                   |
| close                     | double                   |
| volume                    | double                   |
| quote_volume              | double                   |
| count                     | bigint                   |
| taker_buy_volume          | double                   |
| taker_buy_quote_volume    | double                   |

Use raw data to build features for the model (tokenize OHLCV data) and for backtesting.

Data need to be split into three splits:
- 80% training data
- 10% validation data
- 10% test data - used for backtesting and analysis

## Code

Scripts:
- `scripts/prepare.py` - generates required data from the raw data
- `scripts/train.py` - finetunes the model using token sequences
- `scripts/test.py` - uses model to generate signals and backtest the strategy based on signals generated by the model

Run each script using `uv run` command:
- `uv run scripts/prepare.py` to prepare data and save the data required for finetuning and testging
- `uv run scripts/train.py` to run finetuning of a model checkpoint using prepared data
- `uv run scripts/test.py` to perform backtesting using test data split.

Edit each script to modify the process. Try to edit only a single file per experiment if it's possible - i.e. either change tokenization (data preparation) process or data format or change finetuning settings or algorithm.
